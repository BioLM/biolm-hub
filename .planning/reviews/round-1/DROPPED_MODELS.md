# Dropped models (Round-1 decisions, 2026-06-30)

Three models were removed from the catalog as a result of the Round-1 review. Recorded here so the
reasons — and the conditions for bringing the temporary ones back — are not lost. (History is nuked at
launch, so "re-include" means re-port from the internal source or revert the drop commit, then fix.)

## `clean` — PERMANENT drop
**Reason:** upstream CLEAN ships a **Non-Exclusive Research Use License** (non-commercial). The model dir
asserted BSD-3 (and linked a 404 license path), which is incorrect. A non-commercial license is
incompatible with this catalog's permissive, commercial-friendly inclusion gate.
**Re-include only if:** upstream relicenses under a permissive license (MIT/Apache/BSD), **or** a
permissively-licensed equivalent EC-number / enzyme-function predictor is substituted.

## `boltz` — TEMPORARY drop
**Reason:** the `msa_search` feature depends on `msa_search_nim` — a **closed NVIDIA NIM** that is already
on the permanent EXCLUDE list. Shipping boltz as-is leaks a closed, unavailable dependency into the OSS
repo (the field/code path references a model external users cannot run). The rest of boltz is permissively
licensed and was deploy-proven on dev.
**Re-include when:** the `msa_search` feature is either (a) removed entirely (drop the field + the
`app.py` code path + schema/docs), or (b) replaced with an open MSA path (caller-supplied MSA, or an
open MSA server) — then boltz returns with no closed dependency. **Low bar** — mostly excising the
NIM-coupled path. (See `.planning/reviews/round-1/models/boltz.md`.)

## `rfd3` — TEMPORARY drop
**Reason:** multiple **documented and tested** request parameters — `symmetry`, `cyclic_chains`,
`conditioning_mode`, `output_format`, `smiles`/`ccd_code`, `bonds` — are accepted by the schema but
silently dropped before reaching the design engine (`_create_design_specification`). The public contract
lies: callers set parameters that do nothing, and tests assert on params that aren't wired.
**Re-include when:** those parameters are either (a) wired through to the RFdiffusion engine and verified
on a deploy, or (b) removed from the schema/docs/tests so the contract is honest — then rfd3 returns.
(See `.planning/reviews/round-1/models/rfd3.md`.)

## `esmstabp` — TEMPORARY drop
**Reason:** esmstabp is a **self-trained RandomForest with no public source** — the weights can't be
redistributed or regenerated by an external user (no public training data/pipeline), and it was a bespoke
model hand-held through to internal production. It isn't packaged for open-source reproducibility yet, and
on an empty public bucket it can't deploy at all. (It also carried a 🔴 de-internalization bug —
`_train.py` hardcoded `R2_BUCKET="biolm-modal"` — now moot.)
**Re-include when:** the model can be obtained/reproduced by an external user — i.e. the training data +
pipeline are publishable, **or** the trained weights are hosted in the public bucket under a clear
license with a documented source, so it self-populates like the other weight models.

---

## `peptides` — TEMPORARY drop (2026-06-30, session `oss-w3b-wsec`)
**Reason:** the wrapped PyPI package `peptides==0.3.4` (althonos/peptides.py) is **self-contradictory on
license** — its source + bundled `COPYING` declare **GPL-3.0** (copyleft) while its PyPI metadata says
**MIT**. Shipping it would either put the catalog's only copyleft dependency into an otherwise cleanly-
permissive repo (conservative GPL-3.0 reading) or require an upstream maintainer ping to confirm MIT before
launch — an unresolved-license asterisk on a non-marquee, pure-compute descriptor wrapper. Per user
direction, dropped from v1 rather than carry the headache.
**Re-include when:** upstream (althonos/Martin Larralde) confirms the package is really MIT (then it ships
permissively), **or** the team accepts shipping one GPL-3.0 model and vendors the full GPL-3.0 text into
`models/peptides/LICENSE`. The model code itself is sound (single-variant `encode`, CPU, no weights) — a
clean revert of the drop commit once the license is resolved.

gateway/catalog references, docs counts, shared-asset users) is handled in the Round-1 fix campaign.
